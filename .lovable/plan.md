## بوابة الهيئة التدريسية — خطة العمل

### 1. ربط الأستاذ بالكلية والمقررات
- إضافة حقل `kuliya_id` إلى جدول `profiles` (الكلية التابع لها الأستاذ).
- إضافة حقل `kuliya_id` إلى جدول `courses` (يُملأ تلقائياً من كلية الأستاذ عند إنشاء المقرر).
- صفحة "مقرراتي" تبقى كما هي + بطاقة جانبية تعرض اسم الكلية تلقائياً.
- في `AdminUsers`: يمكن للأدمن تعيين كلية لكل أستاذ.

### 2. كتالوج المقررات للطلاب (انضمام تلقائي)
- صفحة `/portal/student/catalog`: تعرض المقررات مجمّعة حسب الكلية، مع فلتر.
- زر "انضم للمقرر" → ينشئ سطراً في `enrollments` بحالة `active` مباشرة (بدون موافقة).
- المقرر يجب أن يكون `is_open_for_enrollment = true`.
- صفحة `/portal/student/courses`: مقرراتي مع زر مغادرة.

### 3. صفحة تفاصيل المقرر (للطالب وللأستاذ)
- `/portal/course/:id` تعرض:
  - معلومات المقرر والأستاذ والكلية.
  - المحاضرات والمواد (lecture_materials الحالي).
  - **الاجتماعات (Google Meet)** — جديد.
  - **تسجيلات المحاضرات الصوتية** — جديد.
  - الدرجات الخاصة بالطالب (للطالب فقط).

### 4. اجتماعات Google Meet
جدول جديد `course_meetings`:
- `course_id`, `title`, `meet_url`, `scheduled_at`, `status` (scheduled/live/ended), `started_at`, `ended_at`.
- صفحة جديدة في بوابة الأستاذ: `/portal/teacher/meetings` لإنشاء/جدولة اجتماع وإدراج رابط Meet.
- الطلاب المسجلون في المقرر يرون الرابط ويضغطون "انضمام".

### 5. تسجيل صوت المحاضرة من متصفح الأستاذ
عند ضغط الأستاذ "ابدأ المحاضرة + سجّل":
1. يفتح Google Meet في تبويب جديد.
2. صفحتنا تطلب إذنين:
   - **الميكروفون** (`getUserMedia`).
   - **مشاركة تبويب Meet مع الصوت** (`getDisplayMedia({audio:true})`) — لالتقاط أصوات الطلاب.
3. ندمج المصدرين عبر `AudioContext` ونسجل بـ `MediaRecorder` بصيغة `audio/webm; codecs=opus` (حجم صغير جداً ≈ 30MB/ساعة).
4. التسجيل يتم على شرائح (chunks كل 30 ثانية) ويُرفع تدريجياً (resumable upload) إلى bucket جديد `lecture-recordings` (خاص) ليتفادى مشكلة الحجم وانقطاع الإنترنت.
5. عند الإيقاف: دمج الشرائح في سيرفر edge function، حفظ المسار في جدول `lecture_recordings`.
6. الطلاب يستمعون عبر signed URL.

ملاحظات تقنية مهمة للأستاذ:
- يجب أن يبقى تبويب الموقع مفتوحاً طوال المحاضرة.
- عند مشاركة التبويب، يختار "تبويب Meet" ويفعّل "مشاركة صوت التبويب".
- يعمل على Chrome/Edge فقط (Firefox/Safari لا يدعمان مشاركة صوت التبويب).

### 6. التخزين والحجم
- `audio/webm opus` بـ 32kbps mono ≈ **15MB/ساعة** فقط.
- محاضرة 3 ساعات ≈ 45MB → ضمن حدود Lovable Cloud Storage بسهولة.
- لو تجاوزت ساعة، الرفع التدريجي يضمن عدم فقدان البيانات.

### 7. صلاحيات (RLS)
- `enrollments`: الطالب يُنشئ/يحذف سجله بنفسه؛ الأستاذ يرى طلاب مقرراته.
- `course_meetings`: الأستاذ يدير اجتماعات مقرراته؛ الطلاب المسجلون يقرؤون فقط.
- `lecture_recordings`: نفس النمط.
- bucket `lecture-recordings`: خاص، signed URLs.

### 8. ترتيب التنفيذ
1. هجرة قاعدة البيانات (الكليات للأساتذة، الكلية للمقررات، انضمام تلقائي، جداول الاجتماعات والتسجيلات، bucket).
2. صفحة كتالوج الطالب + مقرراتي.
3. صفحة تفاصيل المقرر المشتركة.
4. صفحة اجتماعات الأستاذ + قائمة اجتماعات الطالب.
5. مسجّل الصوت + الرفع التدريجي + edge function للدمج.
6. عرض/تشغيل التسجيلات للطلاب.
